Script describing Database on a Chip

The world is warming. The Guardian reports that we are likely to pass 1.5C in three years from now.

The Wall Street Journal reports that data centers consume 100 times more power than a typical industrial business. According to the International Energy Agency data centers consume about 100 billion kilowatt hours per year, equivalent to the entire usage of a medium sized country like Spain, Mexico, or Britain.

Data centers spend a lot of time doing database look ups using software running on generic CPUs.

We can significantly reduce the amount of energy used to do database look ups by using specialized chips that implement the software currently used to perform database look ups.

Graphics processing units are present in every smart phone these days because they burn one thousand times less power than does the equivalent software implemented on a generic GPU while running one thousand times faster.

Burning less power to do each look up means burning less electricity which means lower electricity bills for data centers and less carbon dioxide in the atmosphere from burning fossil fuels to generate electricity.

We are following the approach outlined in Introduction to VLSI, Mead, Conway, 1980 by designing CPU instructions optimized for database look ups.

Currently we have succeeded in representing the Btree look up algorithm as an instruction implemented in Nand digital logic gates.

We need your help to place this wiring diagram on a real Silicon chip alongside a generic CPU and Ethernet controller so that we can drive these specialized instructions to see if they lead to the expected improvements.

Then we can:

  1. Send the working chip to well known the data centers for evaluation

  2. Report the savings report from one customer keen to persuade ithers that they must follow

  3. Satisfy the demand by having the chip built at scale at a fab in Arizona.

  4. make lots of money while saving the environment!
